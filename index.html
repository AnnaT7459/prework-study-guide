HTML

  • The head element contains information about the webpage.
  • head - metadata (information) helps browser to render the page correctly (not visible)
  • metadata - data about webpage that browser uses to effectively serve webpage to users (contains important info about accessability, search engines, and performance)
  • The body element represents the visible content shown to the user.
  • Hypertext Markup Language - content structure
  • 1 of 3 cornerstone languages used in web development
  • made up of elements that work together to provide basic framework (text, image, header, and list elements)
  • elements to divide content: meta-related, content division, and semantic elements
  • makes it easier to navigate
  • composed with an opening and closing tag - angle brackets (contain what to include in an element)
  • HTML attribute - defines additional info about an element ONLY AT THE BEGINNING
  • title element - important for SED (Search Engine Optimization)
  • SED - process of improving the ranking of a website in Google or Bing
  • style element - defines CSS
  • base element - URL for webpage
  • link element - connects external sources
  • script element - embeds client-side scripts
  • meta element - character set, descriptions, keywords, author, and viewpoint
  • charset element - character encoding
  • http-equiv element - HTTP header
  • viewport element - default view
  • child elements - nested inside other elements
  • parent elements - hold child elements (body elements)
  • semantic HTML elements - name of element describes its content and the role it plays on the webpage
  • div element - page division
  • span element - inline division
  • heading - h1, h2, h3, etc.
  • p element - paragraph/block - wrap text
  • ul element - unordered lists
  • ol element - ordered lists
  • li element - list items
  • img element - info about an image displayed
  • src element - defines location of an image
  • br element - line of empty space (line break) between 2 blocks/paragraphs of content
  • a element - anchor - creates links to other webpages
  • alt attribute/element - text string that describes the appearance and functionality of the image
  • footer element - appears at the bottom of the page an usually includes author, copyright, contact, sitemap, and navigation
  • section elements - semantic - define section in document with thematic content
  • id attribute/element - identify elements to select

CSS

  • CSS - Cascading Style Sheets - programming languagee to define how content is presented to users
  • defines colors, aesthetics, visual layout, and organization
  • style rules
  • 3 ways to style a webpage:(1) inline CSS - styles an element directly in HTML file, quick/permanent, but less flexible than external and internal style sheets (2) internal style sheet - embeds entire style sheet directly into the HTML file in head sections, rules only apply to that page, and configure CSS classes and IDs (3) external style sheet - most common, keeps all CSS rules in a separate file (makes design changes easier)
  • separation of concerns - separates code into different files based on use
  • declarations - 2 components (CSS property and value of property)
  • outside element - margin property (buffer)
  • A margin indicates how much space we want around the outside of an element.
  • inside element - padding property (makes margin property bigger)
  • A padding indicates how much space we want around the content inside an element.
  • DRY - Don't Repeat Yourself code - reuse/share pieces of code to reduce lines, overall size, and complexity of a codebase
  • positioning property - text-align
  • debugging is important - finding mistakes and errors
  • class attribute - allows to share a CSS rule to any element chosen with class selector
  • HTML Color Codes: https://htmlcolorcodes.com/

Git

  • git checkout: -b branch-name: creates a new branch and switches to it
  • branch - a copy or version of the code that can be worked on in isolation
  • MVP = minimum viable product
  • Git is a version control system that allows you to manage and keep track of each version of your project
  • To access - command line
  • Always start with git (for commands)
  • initiates (create) or repo, add code
  • Tracks changes on local computer
  • GitHub:
  • Allows you to collaborate with others.
  • cloud-based hosting service
  • keeps track of projects inside repositories that are stored remotely
  • Git Cheat Sheet: https://education.github.com/git-cheat-sheet-education.pdf
  • repositories can be owned by 1 developer or shared by a team
  • Markdown - lightweight plain-text formatting language (.md)
  • blocker - I'm stuck
  • bottleneck - slowing down production
  • acceptance criteria - set requirements in order to complete a user story "done"
  • feature- functional part of a app/website that has value/fulfills a need in the business
  • Markdown Cheat Sheet: https://www.markdownguide.org/cheat-sheet/
  • IDE- integrated development environment - type of software used by developers that combines features needed to do work in a single interface (graphical user interface - GUI)
  • GUI - a way for users to interact with the computer visually
  • IDE contain tools such as a source code editor, local build automation, a debugger, and ability to extend IDEs capabilities with extensions
  • commit message - description of code change - commit often!
  • pull request (PR) - merges a comparison branch
  • web application - accessed on web browser allows users to perform tasks
  • URL - uniform resource locator
  • www. - world wide web

JavaScript

  • JS: high-level dynamic programming language
  • A variable is a named container that allows us to store data in our code.
  • Control flow and loops are the order in which a computer executes code in a script, flows in a linear way conditionally (if statements)
  • allows users to interact
  • effects interactivity, data communication, and much more
  • Browser Application Proogramming Interfaces (APIs) - functionality of audio, 3D, style, webcam
  • Third Party APIs - functionality of other content such as Facebook
  • Third Party frameworks accelerate work of building
  • Document Object Model (DOM) API - manipulates documents
  • semicolons indicate where a statement ends
  • case sensitive!
  • variables - store values (var)
  • string varibles - textual data enclosed with a single or double quotation marks
  • number variables - no quotation marks needed
  • boolean variables - True/False value (no quotation marks needed)
  • truthy - if statement will do whatever action in between curly brackets; otherwise ignored
  • falsy - Boolean data type (false)
  • operator strict equality - performs testt if values are equal and same data type turns into T/F
  • conditionals - code structures to test if an expression is true or not (if...else)
  • array variables - store multiple values in 1 reference
  • object - value in memory, referenced by an identifier, mutable
  • object literal syntax - limited sets are initialized (can be added or removed)
  • object variables - everything
  • comments - snippets of text that can be added with code
  • operator - mathematical symbol produced from 2 values/variables: addition combines strings, subtraction/multiplication/division self explanatory
  • operator assignments assign a value to a variable
  • operator - not, does-not-equal: returns opposite value of what it precedes, used with equality operator, negation operator, and tests if values are not equal
  • functions - packs functionality you want to reuse, often take arguments: bits of data they need to do their job
  • functions - objects capable of being callable, simple steps of instructions for computer; always followed by (), collection of properties
  • events - code structures that listen for activity in the browser, and run code in response
  • click events are most common
  • arrow function - anonymous function
  • cancel prompts - null: special value that means no value
  • JavaScript Data Types
  • weakly type - allows implicit type conversation when an operation involves mismatched types, instead of throwing type errors
  • primitive values - object define immutable values at lowest level language (can be tested by type of operator; have correspondng object wrapper types)
  • number object - methods like toExponential, stores positive floating point numbers
  • JavaScript wraps values to corresponding wrapper object, accessing property on object
  • undefined type - absence of an object
  • NaN - not a number: operation cannot be expressed by a number (not equal to itself)
  • BigInt type - represents integers with arbitrary magnitue, loosely relate to same mathematical value, no fractions
  • JavaScript strings cannot be modified, but are able to be debugged
  • symbol type - unique, immutable used as the key of an object property guaranteed not to clash with keys from other code
  • symbols - atoms
  • property keys - strings or symbols
  • data property - keey with a value (attributes, values, writable, enumerable, and configurable)
  • accessory property - key with 1 or 2 functions to retrieve/store a value (attributes: get, set, enumerable, and configurable) *DO NOT CONFUSE WITH METHOD*
  • prototype - hidden property of objct
  • arrays - regular objects relatioinshipped between integer-keyed properties and length, ordered lists, group of related data, single variable that holds group
  • typed arrays - underlying binary data buffer, umbrella term
  • index - unique number for inside array location
  • arrays are zero-indexed and sequential
  • loops - predictable pattern of indices to perform a task on all the items in an array - single code to be executed over and over, contains 3 statements (1. starting point for loop 2. condition 3. repeat)
  • JSON - JavaScript Object Notation: lightweight data (interchange format, builds universal data structures - transferrable)
  • type coercion - primitive: no string preference for type; numeric: number and BigInt; string
  • Chrome DevTools - indispensible developer's tool
  • commenting out code - makes inactive without deleting
  • Math - contains static properties and methods for mathematical constants and functions, does not work with BigInt
